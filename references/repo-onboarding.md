# Repo Onboarding

GitHub clone linki verildiğinde şu akışı uygula.

## 1. Başlangıç

- repoyu klonla veya mevcut klasörü incele
- ana klasör yapısını listele
- README, bağımlılık dosyaları ve ana konfigürasyonları incele

## 2. Teknik keşif

En az şunları belirle:

- backend teknolojisi
- frontend teknolojisi
- test altyapısı
- paket yöneticileri
- environment dosyaları / örnekleri
- container veya deploy ipuçları
- mevcut docs var mı

## 3. Uyum analizi

Aşağıdakileri raporla:

- mevcut yapı skill'in hedef klasör mantığına uyuyor mu
- `backend/`, `frontend/`, `tests/`, `docs/` mevcut mu
- farklı klasör yapısı varsa nasıl eşlenecek
- katmanlı mimari izleri var mı
- riskli alanlar neler

## 4. Kullanıcı hedefi netleştirme

Yalnızca kritik boşluklar varsa sor:

- bu repoda hedef yeni özellik mi, bakım mı, refactor mü
- hangi kullanıcı çıktısı isteniyor
- kapsamın dışında tutulması gereken alan var mı

## 5. Plan üretimi

En az şu başlıkları oluştur:

- kısa hedef özeti
- mevcut durum
- hedef durum
- etkilenecek alanlar
- riskler
- görev önerileri
- test beklentisi

## 6. Dağıtım öncesi onay

Her zaman şu soruyla bitir:

**"Plan hazır. Tasklara ayırıp ekibe dağıtayım mı?"**
